# Challenge-execution mode

Use for live competitions/challenges where the primary artifacts are technical outputs rather than a CV.

Primary outputs in `preparation-packet.md`:
- rules/constraints summary
- scoring/evaluation summary
- baseline/SOTA or prior-art notes when available
- project workspace links under `/home/brayan/projects/<workspace>/` when applicable
- experiment/log/reproducibility checklist
- code/README/submission metadata checklist
- final PR/submission review checklist
- compute/cost/risk blockers

Optional companion files:
- `application/submission-checklist.md`
- links to project-local `README`, `EXPERIMENTS.tsv`, `COMMANDER_STATUS.md`, `FEEDBACK.md`, logs, or final candidate artifacts.

Keep runnable code and bulky experiment artifacts outside the vault under `/home/brayan/projects/`; the vault remains the control/documentation layer.

Do not submit public PRs, participant forms, or paid compute actions without Brayan's explicit approval.
